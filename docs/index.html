Base 合约权限执行文档

基于 D:\buzzing 的合约源码和本地文档整理，用于权限迁移规划和验收核对。

1. 权限类型

owner

  • 单一治理地址，可迁移。
  • 通常控制关键配置、目标地址、费用提取和授权名单。

auth / wards

  • 多管理员模型，通过 rely、deny 或 setAuth 管理。
  • 适合返佣发放、结算和后端自动化执行。

oracle

  • 业务结果权限，独立于 owner。
  • 在 PreTrading 中风险较高。

ProxyAdmin.owner

  • 独立控制透明代理升级，不等同于业务 owner。
  • 建议迁移到治理多签，而不是继续留在部署 EOA。

4. 需要确认的事项

  • PreTrading.owner 是否真的迁移。历史文档曾建议只迁移 oracle。
  • ProxyAdmin.owner 的最终多签接收地址目前仍是 TBD。

5. 建议执行顺序

  1. 先迁移 ProxyAdmin.owner。
  2. 再迁移 4 个待处理 owner。
  3. 补齐执行钱包，例如 FeeRebateDistributor.setAuth。
  4. 每一步结束后都从链上读回状态做验收。
