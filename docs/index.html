Base Contract Permissions Ops Doc

Built from D:\buzzing source contracts and local notes for handover planning and verification.

1. Permission types

owner

  • Single governance address. Transferable.
  • Usually controls key config, target addresses, fee withdrawal, and allowlists.

auth / wards

  • Multi-admin model managed by rely, deny, or setAuth.
  • Useful for payouts, settlement, and backend automation.

oracle

  • Business result authority, separate from owner.
  • High risk in PreTrading.

ProxyAdmin.owner

  • Controls transparent proxy upgrades independently of business owner.
  • Best moved to a governance multisig instead of the deployer EOA.
